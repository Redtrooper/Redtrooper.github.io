Project Description

Mechanical Meltdown is a wave-based third-person shooter game built in Unreal Engine 5.4.4 utilizing the Steam Online Subsystem to achieve 4-player networked multiplayer.

About

Mechanical Meltdown was designed as an arcade style shooter drawing inspiration from games such as Call Of Duty: Zombies. The gameplay is designed to be replayable and allow for player customization based on play style. Created in collaboration with developers, artists, musicians, and a game designer this project serves as the current largest that I have led and worked on. The project served as my final project at Full Sail University.

The Combat System

In creating the combat system for Mechanical Meltdown I split the weapon functionality between many different classes. By abstracting the weapons this way, the player only needs to include and work with a "Base Weapon" while the child classes process function calls. Additionally, each weapon could build on the functionality of the last. The "Base" classes were designed as the bare minimum a weapon of that type would need to function (not including any models or animations). For example, any weapon that inherits from "Base Ranged Weapon" will have the ability to shoot some projectile once the type is specified. Each weapon has it's own class to allow for weapon-specific behaviors and upgrades. The best example of these behaviors in game would be the hammer, which has a lightning strike ability and a throw ability making it the most diverse weapon in our game currently. Furthermore, the ranged weapons also have accuracy modifiers (including first-shot accuracy) and support recoil.

Upgrades

Allowing the player to create builds and play the game their way means adding different systems for branching upgrades during gameplay. First, I created the skill tree to add some basic upgrades for the player stats including health, speed, and inventory size. These upgrades alone wouldn't allow for much customization, so I added 2 different abilities that the player can unlock to change up gamepaly. Currently the game has 2 abilities, the dash and the electirc punch. One serves as a combat ability while the other serves as movement, however you may only have one equipped at once. This expands the already existing combat system, while leaning more into player customization.

Next, I created an upgrades system for the weapons that allows for the increasing of weapon stats. The weapon charge station allows the player to upgrade their weapons, and in the future will support the adding of weapon-specific abilities and bonuses.

Multiplayer

The largest achievement in the creation of Mechanical Meltdown was the addition of networked 4-player multiplayer. In being a student project, getting networked multiplayer working was one of our largest goals. Not only did it provide an excellent learning oppurtunity, but better prepared us for the industry as multiplayer is a large portion of many of the best and most popular games released. My portion of the implementation of multiplayer was replicating gameplay features and refactoring existing replication code for better preformance. I focused mostly on weapons and abilities, writing some core In doing so I was able to understand the fundamentals of Player-To-Player networked multiplayer.

Next Steps

In continuing this project, my first goals will be to review the existing code for any issues that would prevent it from being easily readable or expandable. After completing any necessary refactoring, I plan to expand upon the existing features before adding new ones. First, I will complete the weapon upgrades system and allow each weapon to have it's own specialized abilities. Next, I plan to work on the abilities system by providing the player with more information about which ability they have equipped through UI. Afterwards, I will work on the addition of more weapons and abilities while focusing on keeping the core gameplay loop balanced.
